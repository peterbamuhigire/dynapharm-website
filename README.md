# Dynapharm Africa — Marketing Website

A multi-language static marketing website for **Dynapharm Africa**, an international health and wellness company operating in 40+ countries. Built with Astro, Tailwind CSS, and Alpine.js.

## About

This website presents the Dynapharm Africa brand to three audiences:

1. **Health-conscious consumers** seeking natural supplements
2. **Prospective distributors** considering the business opportunity
3. **Existing distributors** sharing a professional brand site with prospects

The site is **franchise-neutral** — it represents the Dynapharm brand globally without promoting any single country, franchise, or pricing.

## Pages

| Page | Path | Description |
|------|------|-------------|
| Home | `/` | Hero, product highlights, business opportunity teaser, testimonials |
| Products | `/products` | Interactive catalogue with 8 categories, Alpine.js filtering and modals |
| Business Opportunity | `/business-opportunity` | MLM model, rank progression (DI → CDM), training, DPCs |
| Wellness Guide | `/wellness-guide` | 11 health categories mapped to product recommendations |
| About | `/about` | Company story, mission, values, brand pillars |
| Contact | `/contact` | Enquiry form (info@dynapharmafrica.com), distributor CTA |
| FAQ | `/faq` | 20+ questions covering products, business, and company |

## Languages

- **English** (en) — default
- **French** (fr) — francophone African markets

URL structure: `/{lang}/page` (e.g., `/en/products`, `/fr/products`)

## Tech Stack

- **Astro** — static site generator, zero JS by default
- **Tailwind CSS v4** — utility-first styling
- **Alpine.js** — lightweight interactivity (product filtering, modals)
- **Fontsource** — self-hosted fonts
- **GSAP** (optional) — scroll-triggered animations

## Build Commands

```bash
npm run dev          # Development server at localhost:4321
npm run build        # Build to dist/
npm run preview      # Preview production build
```

## Deployment

Static site deployed via git pull and build. No server-side runtime, no database, no CMS.

```
Local:  edit → build → git push
Server: git pull → npm run build → nginx serves dist/
```
